# Sabai Haus AutoFlow Wellness OS - Agent Instructions

## Project Identity

This project is a completely separate system from LBL AutoFlow.

Project name:
Sabai Haus AutoFlow Wellness OS

Industry:
Massage, Spa, Wellness, Head Spa, Beauty Wellness

Main goal:
Build an all-in-one management system for wellness businesses, focused on appointment management, customer retention, package tracking, therapist performance, WhatsApp follow-up, and AI growth suggestions.

Do not include bakery, kitchen, mini tart, product recipe, flavour, or food production logic.

---

## Product Positioning

Sabai Haus AutoFlow is not just a CRM.

It should be designed as:

AI Wellness Business Operating System

Core idea:
AoiKumo helps businesses manage operations.
Sabai Haus AutoFlow should help businesses manage, retain, upsell, and grow revenue.

## UX Principles

The system must be easy for non-technical spa staff.

Every page should be:
- Clear
- Visual
- Fast to understand
- Mobile-friendly
- Suitable for receptionist, therapist, and business owner

Important:
Do not overload users with too many tables.
Use cards, badges, status labels, quick actions, and filters.

## AI Growth Features

Prioritize features that help owners make more money:

1. AI Customer Recovery
Find customers who have not returned for 30, 60, or 90 days.

2. AI Upsell Suggestions
Recommend add-on services based on customer history.

3. AI Business Advisor
Turn dashboard data into action suggestions.

4. WhatsApp Follow-Up
Generate professional WhatsApp messages for booking reminders, birthday greetings, package renewal, customer recovery, and after-treatment care.

5. Corporate Wellness CRM
Support future B2B leads for HR, offices, hotels, gyms, and condominiums.

## Coding Rules

Use TypeScript.

Keep components clean and reusable.

Use clear file names.

Avoid hardcoding business data deeply into components.

Use mock data first if database is not ready.

Before making large changes:
- explain what will be changed
- keep existing working structure stable
- avoid breaking the dev server
